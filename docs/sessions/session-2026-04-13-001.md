## Session Notes

- **Purpose field was the key design insight.** Scope describes domain, purpose describes intent. Without purpose, the Captain fills the vacuum with imagined objectives. The gemstones failure pattern (4 consecutive UAT runs) was caused by this gap, not by LLM non-determinism.

- **Compiler plan mode integration enforces the spec's role boundary.** Captain orchestrates scope and sequencing; Compiler makes editorial decisions. The Structuring prompt explicitly says "defer to the Compiler's editorial decisions." This eliminated a whole class of failures where the Captain wrote acceptance criteria the source material couldn't satisfy.

- **Purpose-convergence is Phase 7's job.** Build phase achieves coverage (all sources compiled). Stationed mode achieves purpose convergence via the Explorer gap -> Scout mission feedback loop. This design intent is documented in REQUIREMENTS.md Phase 7 section.

- **Three evaluation passes is normal.** Each pass found real issues. The iterative fix-and-recheck loop caught: substring matching bugs, resource leaks, trace context leaks, missing tests, contradictory prompts. Don't skip re-evaluation.
