## Session Notes

- **Phase branch created:** `phase/4-explorer-interactive` branched from `main` at `e4c925c`
- **Architecture decision:** All embedding providers route through `LiteLLMEmbeddingClient` via the factory. No provider-specific client classes. This keeps the codebase simple and consistent with the litellm-as-provider-agnostic-layer pattern. If a future provider needs SDK-specific features (e.g. true interleaved multimodal), a new client class can be added behind the same ABC.
- **Auth pattern:** Gemini API uses `GEMINI_API_KEY` env var (litellm convention). No explicit auth config needed in workspace config.
- **Default dimensions changed from 384 to 768.** Existing workspaces with `active: ollama` will continue using 384. New workspaces get 768 (gemini default). No migration needed — dimensions are read from config at runtime.
