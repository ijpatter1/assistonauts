## Session Notes

- **UAT was the forcing function.** Running the UAT with real book images (8 PNG pages) exposed 3 systemic failures that unit tests couldn't catch: YAML parser fragility with verbose LLM responses, Captain hallucinating file paths and article types, Curator singleton leaking between sequential tasks. All 3 required understanding the full plan→execute→verify pipeline, not just individual component behavior.
- **Captain needs Sonnet, not Haiku.** Haiku 4.5 cannot reliably produce the structured YAML mission plans the Captain parser expects. Default config now maps captain to claude-sonnet-4-6. Other agents remain on Haiku for cost.
- **parse_plan_response: extract, don't strip.** The original approach (strip code fences, parse full response) breaks when the LLM adds explanatory text. The new approach (regex-extract between fences) is robust to preamble/postamble text.
- **ArticleType correction map pattern.** The `_ARTICLE_TYPE_CORRECTIONS` dict + `_validate_params` correction is a reusable pattern for any place where LLMs produce close-but-invalid enum values. Consider applying to other validated fields in future phases.
- **Agent cleanup via try/finally.** All agents should have a `close()` method for resource cleanup, even if it's a no-op. The TaskRunner now calls it if present. This matters for any singleton or resource-holding agent.
