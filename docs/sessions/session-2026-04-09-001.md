### Priority 1: Gemini Embedding 2 Integration (Level 2 — Multimodal)

**Goal:** Replace the current `LiteLLMEmbeddingClient` (ollama/nomic-embed-text) with a `GeminiEmbeddingClient` that supports multimodal embeddings via the `google-genai` SDK.

**Why Gemini Embedding 2:**

- Best-in-class multimodal: natively embeds text, images, PDFs, audio, and video
- 8192-token input context — most wiki articles fit without chunking
- Interleaved input — can embed image+text together in a single request
- Directly aligns with Scout's existing image ingestion pipeline

**Implementation plan:**

1. **Add `google-genai` dependency** to `pyproject.toml` — medium
2. **Create `GeminiEmbeddingClient(EmbeddingClient)`** in `src/assistonauts/archivist/embeddings.py` — medium
   - Text-only `embed(text)` and `embed_batch(texts)` for drop-in compatibility
   - New `embed_content(data: bytes, mime_type: str) -> list[float]` method for multimodal
   - New `embed_multimodal(parts: list[dict]) -> list[float]` for interleaved input (e.g., image + caption together)
   - Uses `client.models.embed_content(model="gemini-embedding-2-preview", contents=...)`
   - Authentication via GCP service account (already provisioned)
3. **Extend `EmbeddingClient` ABC** — small
   - Add optional `embed_content(data: bytes, mime_type: str)` with a default `NotImplementedError` so existing implementations aren't broken
   - Update `FakeEmbeddingClient` in `tests/helpers.py` to support the new method
4. **Update config model** — small
   - Add `gemini` as a provider option in `EmbeddingProviderConfig`
   - May need `api_key_env` or `service_account_path` field for auth
   - Update `embedding_dimensions` default for Gemini (check actual output dims)
5. **Wire into Archivist indexing** — medium
   - `index_with_embeddings()` should detect image sources and use `embed_content()` for them
   - Text articles continue using `embed(text)`
   - Scout-ingested images get their own embedding alongside the text extraction
6. **Update CLI** — small
   - `assistonauts index` and `assistonauts curate` should read provider from config and instantiate the right client
   - Fix the hardcoded defaults in `curate.py`
7. **Contract tests** — medium
   - Test `GeminiEmbeddingClient` with recorded/mocked responses
   - Test multimodal embedding path with image fixtures
8. **Update `config.yaml` schema** — small
   ```yaml
   embedding:
     active: gemini
     providers:
       gemini:
         model: gemini-embedding-2-preview
         # auth via ADC (service account) — no API key needed
   ```

**SDK usage reference** (from user):

```python
from google import genai
from google.genai import types

client = genai.Client()  # or genai.Client(vertexai=True, project=PROJECT_ID, location='us-central1')

result = client.models.embed_content(
    model="gemini-embedding-2-preview",
    contents=[
        "text input",
        types.Part.from_bytes(data=image_bytes, mime_type="image/png"),
    ],
)
embeddings = result.embeddings  # list of embedding vectors
```
