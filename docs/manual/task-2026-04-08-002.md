## Context

Phase 3 implemented the full build pipeline: Scout (with image ingestion via vision model), Compiler, Archivist (indexing + FTS + embeddings), and Curator (cross-referencing). All components have unit tests with fake LLM clients, but the pipeline has never been tested end-to-end with real content and a real LLM.

The test material is _There's Treasure Inside_ by Jon Collins-Black — a treasure hunt book with 23 chapters covering historical treasures, collectibles, and clues. The input is 8 page-spread screenshots from the book.

**Depends on:** `task-2026-04-08-001.md` (branch push + PR merge) must be completed first.

**Bugs found during testing:**

- litellm's Ollama provider requires `Pillow` for image processing. Fixed in commit `162158a`.
- Scout CLI wasn't reading workspace config — vision model config was ignored. Fixed in commit `a3658a3`.
- Gemma 4 (both e2b and 31b) can't handle vision/transcription of book pages. Switched scout role to Claude Haiku 4.5.

## Steps

### Prerequisites

```bash
# Ensure Ollama is running with an embedding model
ollama pull gemma4:e2b
ollama pull nomic-embed-text
ollama serve  # if not already running

# Ensure Pillow and deps are installed
pip install Pillow

# Ensure ANTHROPIC_API_KEY is set (Haiku for vision)
export ANTHROPIC_API_KEY="your-key-here"
```

### 1. Initialize a test workspace

```bash
assistonauts init /tmp/test-kb
```

Then configure scout to use Haiku for vision extraction:

```bash
cat > /tmp/test-kb/.assistonauts/config.yaml << 'EOF'
llm:
  providers:
    ollama:
      model: ollama/gemma4:e2b
      base_url: http://localhost:11434
    anthropic:
      model: claude-haiku-4-5
      api_key_env: ANTHROPIC_API_KEY
  roles:
    scout: anthropic
    compiler: anthropic
    curator: anthropic
    captain: anthropic
    inspector: anthropic
    explorer: anthropic

embedding:
  active: ollama
  providers:
    ollama:
      model: nomic-embed-text
      base_url: http://localhost:11434

cache:
  llm_responses:
    enabled: true
    backend: sqlite
    ttl_hours: 168
    max_size_mb: 500
EOF
```

- Expected result: Workspace created. Config routes scout to Anthropic/Haiku, everything else to Ollama.

- Expected result: 8 markdown files in `/tmp/test-kb/raw/articles/` with YAML frontmatter followed by extracted text.
- **Check quality:** Open a few `.md` files and verify the vision model produced reasonable text.

### 3. Compile each source into wiki articles

The book has a clear structure. Map each page spread to an appropriate article:

