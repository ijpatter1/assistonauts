## Context

Phase 3 implemented the full build pipeline: Scout (with image ingestion via vision model), Compiler, Archivist (indexing + FTS + embeddings), and Curator (cross-referencing). All components have unit tests with fake LLM clients, but the pipeline has never been tested end-to-end with real content and a real LLM.

Image ingestion sends base64-encoded images to a vision-capable LLM (Gemma 4 via Ollama) using the standard multimodal content blocks format. The extracted text flows through the same pipeline as any other source.

**Depends on:** `task-2026-04-08-001.md` (branch push + PR merge) must be completed first.

## Steps

### Prerequisites

```bash
# Ensure Ollama is running with Gemma 4 and an embedding model
ollama pull gemma4:e2b
ollama pull nomic-embed-text
ollama serve  # if not already running
```

### 1. Initialize a test workspace

```bash
assistonauts init ~/test-kb
cat ~/test-kb/.assistonauts/config.yaml
```

- Expected result: Workspace created. Config shows `ollama/gemma4:e2b` as default model, `nomic-embed-text` for embeddings.

### 2. Ingest book screenshots

Pick 5-10 screenshots from the book. Ingest each one:

```bash
assistonauts scout ingest ~/path/to/page-01.png -w ~/test-kb
assistonauts scout ingest ~/path/to/page-02.png -w ~/test-kb
assistonauts scout ingest ~/path/to/page-03.png -w ~/test-kb
# ... repeat for each page
```

- Expected result: Each page produces a markdown file in `~/test-kb/raw/articles/page-NN.md` with YAML frontmatter (`source:`, `ingested_by: scout`, `ingested_at:`) followed by extracted text.
- **Check quality:** Open a few `.md` files and verify the vision model produced reasonable text. Note any issues (garbled text, missed content, formatting problems).

### 3. Compile each source into wiki articles

For each ingested source, compile it into a structured article:

```bash
# List what was ingested
ls ~/test-kb/raw/articles/

# Compile each (adjust title and article-type as appropriate)
assistonauts mission run --agent compiler \
  --source ~/test-kb/raw/articles/page-01.md \
  --title "Chapter 1 - Introduction" \
  --article-type concept \
  -w ~/test-kb

assistonauts mission run --agent compiler \
  --source ~/test-kb/raw/articles/page-02.md \
  --title "Chapter 2 - Fundamentals" \
  --article-type concept \
  -w ~/test-kb

# ... repeat for each source
```

- Expected result: Articles appear in `~/test-kb/wiki/concept/` with proper frontmatter, section structure, and content summaries (`.summary.json` files alongside each article).
- **Check quality:** Verify articles have the schema sections (Key Concepts, Sources, etc.) filled in with actual content from the source.

### 4. Index all articles

```bash
assistonauts index -w ~/test-kb
```

- Expected result: All wiki articles indexed. Output shows "Indexed: N" for each article. If embedding model is configured and running, should show "FTS + embeddings" mode.
- If you see "FTS only", check that `nomic-embed-text` is available: `ollama list`

### 5. Check status

```bash
assistonauts status -w ~/test-kb
```

- Expected result: Table showing wiki article count, raw source count, indexed count matching, word count > 0.

### 6. Run structural analysis

```bash
assistonauts curate --proposals -w ~/test-kb
```

- Expected result: Graph metrics (articles, links, density). Likely shows all articles as orphans (no cross-references yet — expected, since articles were compiled independently).

### 7. (Optional) Inspect the database directly

```bash
sqlite3 ~/test-kb/index/assistonauts.db "SELECT path, title, word_count FROM articles;"
sqlite3 ~/test-kb/index/assistonauts.db "SELECT path, retrieval_keywords FROM summaries;"
```

- Expected result: All articles present with reasonable word counts and keyword extractions.
