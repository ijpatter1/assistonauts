## Context

Phase 3 implemented the full build pipeline including features discovered during initial testing: batch ingestion, image auto-resize, multi-source compilation. This test validates the complete chain with real book content.

The test material is _There's Treasure Inside_ by Jon Collins-Black — a treasure hunt book. The input is 8 page-spread screenshots (each image contains two book pages side by side).

**Book structure (from Table of Contents):**

- Cover (1 spread)
- Front matter: copyright, dedication (1 spread)
- Table of Contents (1 spread)
- Introduction: pages 7-17 (5 spreads: page-008-009 through page-016-017)

All 5 page spreads from pages 8-17 are the Introduction — Chapter 1 doesn't start until page 18.

**Bugs found and fixed during earlier testing:**

- Pillow required by litellm for Ollama images (commit `162158a`)
- Scout CLI not reading workspace config (commit `a3658a3`)
- Images >5MB rejected by Anthropic API — auto-resize added (commit `69d420b`)
- Gemma 4 can't handle vision transcription — use Haiku 4.5 instead
- Improved transcription prompt for book pages (commit `e2a8165`)

## Steps

### Prerequisites

```bash
# Ensure Ollama is running with an embedding model
ollama pull nomic-embed-text
ollama serve  # if not already running

# Ensure deps are installed
pip install -e ".[dev]"

# Ensure ANTHROPIC_API_KEY is set
export ANTHROPIC_API_KEY="your-key-here"
```

### 1. Initialize workspace and configure providers

```bash
rm -rf /tmp/test-kb
assistonauts init /tmp/test-kb

cat > /tmp/test-kb/.assistonauts/config.yaml << 'EOF'
llm:
  providers:
    anthropic:
      model: claude-haiku-4-5-20251001
      api_key_env: ANTHROPIC_API_KEY
  roles:
    scout: anthropic
    compiler: anthropic
    curator: anthropic
    captain: anthropic
    inspector: anthropic
    explorer: anthropic

embedding:
  active: ollama
  providers:
    ollama:
      model: nomic-embed-text
      base_url: http://localhost:11434

cache:
  llm_responses:
    enabled: true
    backend: sqlite
    ttl_hours: 168
    max_size_mb: 500
EOF
```

- Expected result: Workspace created with Haiku for all LLM roles, Ollama for embeddings.

### 2. Batch ingest all page screenshots

```bash
assistonauts scout ingest \
  input_artifacts/cover.png \
  input_artifacts/front-01-02.png \
  input_artifacts/front-03-04.png \
  input_artifacts/page-008-009.png \
  input_artifacts/page-010-011.png \
  input_artifacts/page-012-013.png \
  input_artifacts/page-014-015.png \
  input_artifacts/page-016-017.png \
  -w /tmp/test-kb
```

- Expected result: 8 green checkmarks, one per file. Markdown files created in `/tmp/test-kb/raw/articles/`.
- Large images (front-01-02.png ~10MB, front-03-04.png ~11MB) should auto-resize without errors.
- **Check quality:** `cat /tmp/test-kb/raw/articles/page-008-009.md` — should contain readable transcription of the Introduction opening ("I've hidden a treasure somewhere in the United States...").

### 3. Compile articles

The book has three logical groupings in our test material:

**3a. Cover — single source, entity type**

```bash
assistonauts mission run --agent compiler \
  --source /tmp/test-kb/raw/articles/cover.md \
  --title "Theres Treasure Inside by Jon Collins-Black" \
  --article-type entity \
  -w /tmp/test-kb
```

**3b. Front matter — single source, log type**

```bash
assistonauts mission run --agent compiler \
  --source /tmp/test-kb/raw/articles/front-01-02.md \
  --title "Publication and Dedication" \
  --article-type log \
  -w /tmp/test-kb
```

**3c. Table of Contents — single source, log type**

```bash
assistonauts mission run --agent compiler \
  --source /tmp/test-kb/raw/articles/front-03-04.md \
  --title "Table of Contents" \
  --article-type log \
  -w /tmp/test-kb
```

**3d. Introduction — MULTI-SOURCE compilation (5 page spreads → 1 article)**

This is the key test for multi-source compilation. All 5 page spreads are the Introduction (pages 7-17):

```bash
assistonauts mission run --agent compiler \
  --source /tmp/test-kb/raw/articles/page-008-009.md \
  --source /tmp/test-kb/raw/articles/page-010-011.md \
  --source /tmp/test-kb/raw/articles/page-012-013.md \
  --source /tmp/test-kb/raw/articles/page-014-015.md \
  --source /tmp/test-kb/raw/articles/page-016-017.md \
  --title "Introduction" \
  --article-type concept \
  -w /tmp/test-kb
```

- Expected result: 4 articles total:
  - `/tmp/test-kb/wiki/entity/theres-treasure-inside-by-jon-collins-black.md`
  - `/tmp/test-kb/wiki/log/publication-and-dedication.md`
  - `/tmp/test-kb/wiki/log/table-of-contents.md`
  - `/tmp/test-kb/wiki/concept/introduction.md` (compiled from 5 sources)
- Each article should have a `.summary.json` file alongside it.
- **Check multi-source quality:** `cat /tmp/test-kb/wiki/concept/introduction.md` — should be a coherent article synthesized from all 5 page spreads, with all sources listed in frontmatter.
